服务器返回的数据结构规范如下

{
    "code":200,//返回结果状态码
    "message": "成功",//返回结果说明信息
    "data":{
       
    }//返回结果的数据，是一个map集合
}

目前服务器定义的状态码

200L, "success"
300L, "System Business Error."
330L, "参数校验失败"
402L, "用户名已经注册"
403L, "注册失败"
500L, "登录失败"
600L, "查询文章结果为空"
601L, "文章不存在"
700L, "Token校验失败"
800L, "无收藏的文章"
900L, "图片上传失败"
901L, "图片不存在"
1000L, "压缩包上传失败"
1001L, "压缩包不存在"


特殊说明
安全考虑，服务器和客户端的交互不直接暴露用户ID，而是使用open_id，access_token进行交互，这三个参数能够唯一确定一个用户ID，用户注册或者登录以后都会拿到这三个参数，access_token是有时效性的，一定时间会过期，过期之后客户端需要重新登录申请访问凭证，过期时间是可以在客户端配置的，目前是3个月，类似QQ的登录方式。

接口列表
1、注册接口
接口地址： /account/register (如：https://121.42.44.23/account/register)
参数列表
deviceId:设备ID
token:设备的push token
返回数据

{
    "code": 200,
    "message": "成功",
    "data": {
        "user": {
            "open_id":open_id,
            "access_token":access_token
        }
    }
}

3、文章列表

page：页码

open_id：OpenID 登录之后带上 不登录无需传给服务器
access_token：访问凭证 登录之后带上 不登录无需传给服务器

    "content": "",
    "avatar": 0,

7、文章压缩包下载接口
接口地址： /gzip/${gzipId} (如：https://121.42.44.23/gzip/3)
参数列表
无
返回数据

如果压缩包不存在就返回
{
    "code": 1001,
    "message": " 压缩包不存在",
    "data": {
    }
}

如果压缩包存在就是正常的文件下载，服务器会将文件流返回
